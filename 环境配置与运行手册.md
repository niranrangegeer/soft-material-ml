## 硬件要求

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| GPU 显存 | 4 GB (batch=1, FP16) | 8 GB+ (batch=4+) |
| 内存 RAM | 8 GB | 16 GB+ |
| 磁盘 | 10 GB 空闲 | 50 GB 空闲 |
| 操作系统 | Ubuntu 22.04+ / WSL2 | Ubuntu 24.04 |

## 第一部分：Windows 笔记本配置 (WSL2)

> **为什么要 WSL2？** TensorFlow 2.11 起不再支持原生 Windows GPU。WSL2 是微软官方推荐的方案：在 Windows 内运行一个轻量 Linux 虚拟机，GPU 直通，性能接近原生 Linux。

### 1.1 安装 WSL2

**第 1 步**：右键开始菜单 → **Windows PowerShell (管理员)**

**第 2 步**：输入以下命令并回车：

```powershell
wsl --install
```

**第 3 步**：等待下载完成（约 2~5 分钟），然后**重启电脑**。

### 1.2 安装 Ubuntu

重启后，可能会自动弹出一个 Ubuntu 窗口。如果没有，打开 PowerShell (管理员)：

```powershell
wsl --install -d Ubuntu
```

首次启动 Ubuntu 时，会要求设置用户名和密码：

```
Enter new UNIX username: ubuntu
New password: ubuntu
Retype new password: ubuntu
```

> 注意：输入密码时屏幕不会显示任何字符，这是正常的。记住你设的用户名和密码。

### 1.3 配置 Python 环境

打开 Ubuntu 终端（开始菜单搜索 "Ubuntu"），逐行执行：

```bash
# 更新系统包列表
sudo apt update

# 安装 Python 虚拟环境组件
sudo apt install -y python3.12-venv

# 创建虚拟环境 (命名为 tf-env)
python3 -m venv ~/tf-env

# 激活虚拟环境 (每次打开 Ubuntu 都要先执行这一行)
source ~/tf-env/bin/activate
```

执行完 `source` 命令后，终端提示符前会出现 `(tf-env)`，说明虚拟环境已激活。

> 之后每次重新打开 Ubuntu 终端，都要先执行：`source ~/tf-env/bin/activate`

### 1.4 安装 TensorFlow

继续在 Ubuntu 中（确保前面 `(tf-env)` 已显示）：

```bash
pip install tensorflow matplotlib numpy
```

等 5~10 分钟，看到 `Successfully installed` 就完成了。

### 1.5 导入项目代码

```bash
# 从 Windows 目录复制到 WSL (替换为你的实际路径)
cp -r "/mnt/c/Users/28679/Desktop/机器学习+软材料 科研训练/机器学习+软材料 科研训练" ~/soft-material-ml

# 进入项目目录
cd ~/soft-material-ml
```

## 第二部分：Linux 服务器配置

> 本节适用于 Ubuntu 22.04/24.04 服务器（如实验室 RTX 3090 机器）。

### 2.1 安装 NVIDIA 驱动

```bash
# 检查是否已有 NVIDIA 驱动
nvidia-smi
```

如果显示 GPU 信息和 Driver Version，说明驱动已安装，**跳到 2.2**。

如果没有，安装驱动：

```bash
# Ubuntu 24.04 系统推荐
sudo apt update
sudo apt install -y nvidia-driver-550
sudo reboot
```

重启后再次执行 `nvidia-smi` 确认驱动正常。

### 2.2 安装 CUDA Toolkit

TensorFlow 2.21 需要 CUDA 12.5 和 cuDNN 9。使用 pip 安装最快：

```bash
# 先装 Python 虚拟环境工具
sudo apt update
sudo apt install -y python3.12-venv python3-pip
```

### 2.3 安装 Miniconda

如果你的服务器无法用 pip（网络限制或权限问题），推荐用 Miniconda：

```bash
# 下载 Miniconda
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh -O ~/miniconda.sh

# 安装 (一路按 Enter，遇到 yes/no 选 yes)
bash ~/miniconda.sh

# 重新加载 shell 配置
source ~/.bashrc
```

### 2.4 创建环境并安装 TensorFlow

**方案 A：用 venv（推荐，简单）**

```bash
python3 -m venv ~/tf-env
source ~/tf-env/bin/activate
pip install tensorflow matplotlib numpy
```

**方案 B：用 Conda**

```bash
conda create -n tf-env python=3.12 -y
conda activate tf-env
pip install tensorflow matplotlib numpy
```

无论哪种方案，安装完成后验证：

```bash
python3 -c "import tensorflow as tf; print(tf.config.list_physical_devices('GPU'))"
```

应该输出类似 `[PhysicalDevice(name='/physical_device:GPU:0', device_type='GPU')]`，说明 GPU 可用。

> 如果输出 `[]`，说明 GPU 不可用。跳到 [第五部分：常用问题排查](#第五部分常用问题排查)。

### 2.5 导入项目代码

```bash
# 方式1：从 GitHub 下载 (推荐)
git clone https://github.com/niranrangegeer/soft-material-ml.git ~/soft-material-ml

# 方式2：从 U 盘拷贝
# cp -r /path/to/usb/soft-material-ml ~/soft-material-ml

cd ~/soft-material-ml
ls
```

应该看到 `Main code.py`、`Train_images/`、`Test_images/` 等文件。

## 第三部分：运行训练

### 3.1 激活环境

每次新开终端都要先激活：

```bash
# 如果用 venv
source ~/tf-env/bin/activate

# 如果用 conda
conda activate tf-env
```

提示符出现 `(tf-env)` 即为成功。

### 3.2 验证 GPU 可用

```bash
python3 -c "import tensorflow as tf; print('GPU:', tf.config.list_physical_devices('GPU'))"
```

### 3.3 启动训练

```bash
cd ~/soft-material-ml
python3 "Main code.py"
```

### 3.4 训练过程中会看到什么

```

TensorFlow: 2.21.0
GPU: NVIDIA GeForce RTX 3090 | 显存: 24.0 GB
混合精度(FP16): 已启用 (显存 < 6GB，自动减半)

测试图片: 400 张

  BATCH_SIZE = 1
  混合精度  = FP16

  测试样本  = 400 张

```

每个 `.` = 1 个训练步（1 张图）。每 400 步打印一次进度标记 `[400]`、`[800]`等。每行末尾显示该 epoch 耗时和 loss 值。

### 3.5 中断和恢复

- **停止**：按 `Ctrl+C`
- **恢复**：重新运行 `python3 "Main code.py"`，程序自动从 `Loss.txt` 读取上次进度，从 `SaveModel/` 加载权重继续训练

### 3.6 服务器专用：后台运行

服务器上不能让程序随着 SSH 断开而中断。用 `nohup` 或 `tmux`：

**方式 A：nohup（一行命令）**

```bash
cd ~/soft-material-ml
source ~/tf-env/bin/activate
nohup python3 "Main code.py" > train.log 2>&1 &
```

查看进度：`tail -f ~/soft-material-ml/train.log`

**方式 B：tmux（推荐，可随时连接查看）**

```bash
# 创建新会话
tmux new -s training

# 在 tmux 内启动训练
cd ~/soft-material-ml
source ~/tf-env/bin/activate
python3 "Main code.py"

# 断开 tmux (程序继续运行): 按 Ctrl+B 然后按 D
# 重新连接: tmux attach -t training
```

## 第四部分：迁移到服务器

从笔记本迁移到服务器只需复制 3 样东西：

### 4.1 需要复制的文件

| 文件/目录 | 内容 | 大小 |
|-----------|------|------|
| `Main code.py` | 训练主程序 | ~15 KB |
| `Train_images/Train_images/` | 1600 张训练图片 | ~231 MB |
| `Test_images/Test_images/` | 400 张测试图片 | ~100 MB |

### 4.2 复制方式

**方式 A：用 GitHub（推荐）**

```bash
# 在服务器上
git clone https://github.com/niranrangegeer/soft-material-ml.git ~/soft-material-ml
```

注意：图片文件不在 GitHub 上（太大了），需要单独拷贝 `Train_images/` 和 `Test_images/` 目录，保持目录结构：

```
~/soft-material-ml/
├── Main code.py
├── Train_images/
│   └── Train_images/
│       └── Train_1.jpg ... Train_1600.jpg
└── Test_images/
    └── Test_images/
        └── Test_1.jpg ... Test_400.jpg
```

**方式 B：U 盘/移动硬盘**

1. 在 Windows 上把整个项目文件夹复制到 U 盘
2. 服务器上插 U 盘，挂载后 `cp -r` 到 `~/soft-material-ml/`

**方式 C：scp 远程传输**

```bash
# 在笔记本的 Ubuntu (WSL) 中执行
scp -r ~/soft-material-ml/Train_images username@server_ip:~/soft-material-ml/
scp -r ~/soft-material-ml/Test_images username@server_ip:~/soft-material-ml/
```

### 4.3 迁移后必须做的事

```bash
cd ~/soft-material-ml
# 删除笔记本上生成的检查点（从头开始训练）
rm -rf SaveModel/ Out_rock/ Loss.txt
# 重新创建输出目录
mkdir -p SaveModel Out_rock
```

## 第五部分：常用问题排查

### Q1: `python3: command not found`

```bash
sudo apt install -y python3
```

### Q2: `pip: command not found`

```bash
sudo apt install -y python3-pip
```

### Q3: `ModuleNotFoundError: No module named 'tensorflow'`

说明虚拟环境没激活。执行：

```bash
source ~/tf-env/bin/activate
```

### Q4: GPU 显示 `[]`（GPU 不可用）

**步骤 1**：确认驱动正常
```bash
nvidia-smi
```
如果不显示 GPU 信息，说明驱动没装好，回到 [2.1](#21-安装-nvidia-驱动)。

**步骤 2**：确认 CUDA 库存在
```bash
python3 -c "import tensorflow as tf; print(tf.sysconfig.get_build_info()['cuda_version'])"
```
应该输出 `12.5.1`。

**步骤 3**：如果驱动和 CUDA 版本都正确但仍无 GPU，安装 CUDA Toolkit：
```bash
# 添加 NVIDIA 官方源
wget https://developer.download.nvidia.com/compute/cuda/repos/ubuntu2404/x86_64/cuda-keyring_1.1-1_all.deb
sudo dpkg -i cuda-keyring_1.1-1_all.deb
sudo apt update
sudo apt install -y cuda-toolkit-12-5
sudo reboot
```

### Q5: 训练时提示 `Out of memory` (OOM)

显存不足。编辑 `Main code.py`，将 `BATCH_SIZE = 1` 保持为 1（已经是最小了）。如果还是 OOM：

```python
# 在 Main code.py 中找到 IMG_WIDTH 和 IMG_HEIGHT，改成更小的值
IMG_WIDTH = 128
IMG_HEIGHT = 128
```

这会减少显存占用约 4 倍，但会降低预测质量。

### Q6: matplotlib 报错中文字体

代码已改为英文标题，不会再有字体问题。

### Q7: `No such file or directory` 找不到数据

检查目录结构是否正确：
```bash
ls ~/soft-material-ml/Train_images/Train_images/ | head -5
```
应该看到 `Train_1.jpg` 等文件。如果图片直接在 `Train_images/` 下（没有双层嵌套），编辑 `Main code.py` 将：
```python
PATH_train = './Train_images/Train_images/'
```
改为：
```python
PATH_train = './Train_images/'
```

### Q8: 训练太慢怎么办？

| 调整项 | 位置 | 效果 |
|--------|------|------|
| 减少 EPOCHS | 第 489 行 `EPOCHS = 1001` | 先改成 10 跑通测试 |
| 减小图片尺寸 | 第 88 行 `IMG_WIDTH = 256` | 改成 128 → 4 倍加速 |
| 增加检查点间隔 | 第 350 行 `CHECKPOINT_INTERVAL = 80` | 改成 200 → 少写磁盘 |
| 减少预览图生成 | 第 453 行 `take(1)` 前的条件 | 加 `if epoch % 10 == 0:` |
