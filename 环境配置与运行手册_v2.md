## 0. 你必须先知道的事情

### 笔记本 vs 服务器的分工

| | 笔记本 (RTX 3050 4GB) | 实验室服务器 (RTX 3090 24GB) |
|---|---|---|
| **用途** | 验证代码能跑通 | 实际训练 |
| **能跑 GPU 吗** | 能识别 GPU，但 4GB 显存不够用 | 完全没问题 |
| **建议训练量** | 1~2 个 epoch，确认通路后停止 | 完整 1001 epoch |
| **预计耗时 (1001 epoch)** | 无法完成（OOM 频繁） | 约 2~4 小时 |

### 为什么必须用 Conda

实测结论：**pip 安装 TensorFlow 时，其 nvidia CUDA 库版本经常与 TF 不匹配，导致 GPU 无法识别。** Conda（conda-forge 频道）打包时会自动匹配 CUDA + cuDNN 版本，一步到位。

```
正确做法：conda install -c conda-forge tensorflow
错误做法：pip install tensorflow  (笔记本上实测 CuDA 版本冲突，GPU 不可用)
```

## 第一部分：Windows 笔记本 → WSL2 完整配置

### 1.1 安装 WSL2 + Ubuntu

**Step 1**：右键开始菜单 → **Windows PowerShell (管理员)**，粘贴回车：

```powershell
wsl --install
```

**Step 2**：等待下载（2~5 分钟），然后**重启电脑**。

**Step 3**：重启后开始菜单搜索 "Ubuntu"，打开。首次启动会提示设置用户名和密码。随便设（例如用户名 `ubuntu`，密码 `ubuntu`），密码输入时屏幕不显示字符，这是正常的。

### 1.2 安装 Miniconda（关键！不要用 pip 装 TF）

打开 Ubuntu 终端，逐行执行：

```bash
# 下载 Miniconda 安装脚本
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh -O ~/miniconda.sh

# 安装 (一路按 Enter，遇到 yes/no 输 yes)
bash ~/miniconda.sh

# 初始化 conda
~/miniconda3/bin/conda init bash

# 关掉 Ubuntu 窗口，重新打开
```

### 1.3 创建环境并安装 TensorFlow

重新打开 Ubuntu 后（提示符前缀是 `(base)`）：

```bash
# 接受 conda 服务条款
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/main
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/r

# 创建环境 (python 3.10，兼容性最好)
conda create -n tf-gpu python=3.10 -y

# 安装 TF + CUDA + 依赖 (conda-forge 自动配好 CUDA 版本)
conda install -n tf-gpu -c conda-forge tensorflow matplotlib numpy -y

# 激活环境
conda activate tf-gpu
```

验证 GPU 是否可用：

```bash
python3 -c "import tensorflow as tf; print(tf.config.list_physical_devices('GPU'))"
```

输出包含 `PhysicalDevice(...device_type='GPU')` 即成功。如果输出 `[]`，跳到 [第五部分](#第五部分训练中可能遇到的问题)。

### 1.4 导入项目代码

```bash
# 从 Windows 目录复制到 WSL
cp -r "/mnt/c/Users/28679/Desktop/机器学习+软材料 科研训练/机器学习+软材料 科研训练" ~/soft-material-ml

cd ~/soft-material-ml
```

## 第二部分：Linux 服务器配置（3090 等）

> 服务器是原生 Linux，不需要 WSL。步骤和笔记本几乎一样，但少了"装 WSL"这一步。

### 2.1 确认显卡和驱动

```bash
nvidia-smi
```

输出应显示 GPU 型号和 Driver Version。如果没有，需要先装驱动：

```bash
# Ubuntu 24.04 系统
sudo apt update
sudo apt install -y nvidia-driver-550
sudo reboot
```

### 2.2 安装 Miniconda

```bash
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh -O ~/miniconda.sh
bash ~/miniconda.sh
~/miniconda3/bin/conda init bash
```

关掉终端重新连接（或 `source ~/.bashrc`）。

### 2.3 安装 TensorFlow + CUDA

```bash
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/main
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/r
conda create -n tf-gpu python=3.10 -y
conda install -n tf-gpu -c conda-forge tensorflow matplotlib numpy -y
conda activate tf-gpu

# 验证 GPU
python3 -c "import tensorflow as tf; print(tf.config.list_physical_devices('GPU'))"
```

> 3090 24GB 显存下，可以修改 `Main code.py` 第 115 行 `BATCH_SIZE = 1` 改成 `BATCH_SIZE = 4`，充分利用显存，训练快约 3 倍。

### 2.4 导入代码和数据

**方式 A：从 GitHub 拉代码 + U 盘拷数据**

```bash
git clone https://github.com/niranrangegeer/soft-material-ml.git ~/soft-material-ml
```

注意：图片文件不在 GitHub 上（太大），需要单独把 `Train_images/` 和 `Test_images/` 目录拷贝到 `~/soft-material-ml/` 下，保持目录结构：

```
~/soft-material-ml/
├── Main code.py
├── Train_images/
│   └── Train_images/       ← 1600 张训练图放这里
│       └── Train_1.jpg ... Train_1600.jpg
└── Test_images/
    └── Test_images/        ← 400 张测试图放这里
        └── Test_1.jpg ... Test_400.jpg
```

**方式 B：全量拷贝 (U 盘/移动硬盘)**

把整个项目文件夹拷到 U 盘，服务器上：

```bash
cp -r /media/usb/机器学习+软材料\ 科研训练/机器学习+软材料\ 科研训练 ~/soft-material-ml
```

## 第三部分：运行训练

### 3.1 日常启动流程

每次新开终端，按顺序执行：

```bash
conda activate tf-gpu
cd ~/soft-material-ml
TF_CPP_MIN_LOG_LEVEL=2 python3 "Main code.py"
```

> `TF_CPP_MIN_LOG_LEVEL=2` 只显示 Warning 和 Error，屏蔽 TensorFlow 的冗长编译日志。不加也行，但会刷很多 `inputs=tf.Tensor(...)` 的信息。

### 3.2 首次运行会看到什么

```
Python: 3.10.x
TensorFlow: 2.19.1
GPU: NVIDIA GeForce RTX 3090 | 显存: 24.0 GB
训练图片: 1600 张
测试图片: 400 张
Generator 参数量: 53,804,291
Discriminator 参数量: 2,773,505
未找到检查点，从头开始训练。

============================================================
  BATCH_SIZE = 4
  训练样本  = 1600 张
  检查点间隔 = 每 80 epoch
============================================================

Epoch 0/1001 ................................[400]................................[800].......
```

**首次运行时 `Epoch 0` 会很慢**（1~5 分钟），因为 TF 在编译优化计算图。从 Epoch 1 开始速度恢复正常。

### 3.3 中断与恢复

- **停止训练**：`Ctrl+C`
- **恢复训练**：重新执行 3.1 的启动流程。程序自动从 `Loss.txt` 读取上次进度，从 `SaveModel/` 加载权重继续。

### 3.4 服务器后台运行（不会因断网而中断）

**方法一：nohup（一条命令）**

```bash
conda activate tf-gpu && cd ~/soft-material-ml && nohup python3 "Main code.py" > train.log 2>&1 &
```

查看训练进度：
```bash
tail -f ~/soft-material-ml/train.log
```

**方法二：tmux（推荐，可随时连接进去看）**

```bash
tmux new -s training
conda activate tf-gpu
cd ~/soft-material-ml
TF_CPP_MIN_LOG_LEVEL=2 python3 "Main code.py"

# 断开 tmux（程序继续跑）：先按 Ctrl+B，再按 D
# 重新连入：tmux attach -t training
```

## 第四部分：笔记本 vs 服务器的差异

| 调整项 | 笔记本 (RTX 3050 4GB) | 服务器 (RTX 3090 24GB) |
|--------|----------------------|------------------------|
| BATCH_SIZE | 1（默认） | 4~8 |
| 预计耗时 (1001 epoch) | 无法完成 | 2~4 小时 |
| 检查点磁盘占用 | ~5 GB | ~5 GB（BATCH_SIZE 不影响） |
| 建议用途 | 跑 1 epoch 验证代码通路 | 完整训练 |

**笔记本上用 CPU 验证更快**：

```bash
# 在 Main code.py 中找到 EPOCHS = 1001，临时改成 EPOCHS = 2
# 然后用 CPU 跑（不用 GPU，反而省去显存不足的麻烦）
CUDA_VISIBLE_DEVICES="" python3 "Main code.py"
```

## 第五部分：训练中可能遇到的问题

### Q1: `conda: command not found`

conda 没初始化。执行：
```bash
~/miniconda3/bin/conda init bash
source ~/.bashrc
```

### Q2: GPU 输出 `[]`（GPU 不可用）

**笔记本（WSL2）上**：先确认驱动正常：
```bash
nvidia-smi
```
如果不显示 GPU，说明 WSL2 驱动桥接有问题。管理员 PowerShell 执行：
```powershell
wsl --update
wsl --shutdown
```
然后重新打开 Ubuntu。

**服务器上**：
```bash
nvidia-smi
```
不显示 GPU 则需安装驱动，见 [2.1](#21-确认显卡和驱动)。

### Q3: 训练中途 `Out of memory` 崩溃

减少 `BATCH_SIZE`（Main code.py 第 115 行）或减小图片尺寸（Main code.py 第 88 行 `IMG_WIDTH = 256` → `128`）。

### Q4: 首次运行刷出大量 `inputs=tf.Tensor(...)` 信息

这是 TF 在编译计算图。加 `TF_CPP_MIN_LOG_LEVEL=2` 前缀可屏蔽：
```bash
TF_CPP_MIN_LOG_LEVEL=2 python3 "Main code.py"
```

### Q5: 训练太慢

| 解决方案 | 效果 |
|----------|------|
| 增大 BATCH_SIZE（仅 GPU 显存足够时） | 每个 epoch 快 2~4 倍 |
| 减小图片尺寸 `IMG_WIDTH=128` | 约快 4 倍，但降低精度 |
| 减少 EPOCHS | 直接减少总时间 |

### Q6: 怎么判断训练效果好

1. 看 `Loss.txt`：数字整体趋势向下
2. 看 `Out_rock/epoch_XXXX_1.png`：预测裂纹图和标准答案越来越像
3. 最佳模型保存在 `SaveModel/generator_best.weights.h5`
