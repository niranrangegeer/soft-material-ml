## 1. 这个项目在做什么（一句话版）

> **输入一张多孔材料的几何结构图 + 能量场图，AI 预测出裂纹会从哪里裂开。**

这是一个基于 **U-Net 3+** 架构的 **条件生成对抗网络 (cGAN)**，用于预测随机多孔介质中的不连续裂纹路径。

## 2. 科学背景：多孔介质与裂纹

### 2.1 多孔介质是什么？

就是内部有很多孔洞的固体材料。例如：
- 岩石、混凝土
- 骨骼、木材
- 泡沫金属、陶瓷

### 2.2 为什么要研究裂纹？

多孔材料受到外力时，裂纹会在孔洞之间扩展。预测裂纹路径是材料科学的核心问题——知道材料怎么裂，就能设计出更结实的东西。

### 2.3 传统方法 vs AI 方法

| 传统方法 | AI 方法 |
|---------|--------|
| 有限元分析 (FEM)，逐像素计算应力 | 神经网络直接从图像"学"出裂纹模式 |
| 一个样本跑几小时 | 训练完以后，预测只需几秒 |
| 需要材料学家手动建模 | 给图就行 |

### 2.4 论文来源

这个代码复现的是论文：

> *"Accurate prediction of discontinuous crack paths in random porous media via a generative deep learning"* (2024)

核心创新：用 U-Net 3+（一种先进的图像分割网络）+ GAN（生成对抗网络）来预测裂纹。

## 3. 核心概念词典

### 3.1 深度学习基础

| 术语 | 通俗解释 | 专业定义 |
|------|---------|---------|
| **神经网络 (Neural Network)** | 模仿人脑神经元的多层数学模型 | 由多层可学习参数(权重)构成的计算图，通过反向传播更新 |
| **卷积 (Convolution)** | 用一个小窗口在图片上滑动，提取特征 | 可学习的空间滤波器，对局部感受野做加权求和 |
| **卷积层 (Conv2D)** | 很多个卷积核一起扫图，每个核提取一种特征 | 多通道的二维卷积操作，输出通道数 = 卷积核数 |
| **池化 (Pooling)** | 把图片缩小，保留主要信息 | 下采样操作：MaxPooling取窗口内最大值，降低空间分辨率 |
| **上采样 (UpSampling)** | 把图片放大 | 空间分辨率提升操作：双线性插值或转置卷积 |
| **批次归一化 (BatchNorm)** | 把每一层的数据"标准化"，让训练更稳定 | 对每个mini-batch做均值0方差1的归一化，减少内部协变量偏移 |
| **激活函数** | 给网络加入非线性能力，"弯折"数据 | 非线性变换函数：ReLU(负数变0)、LeakyReLU(负数缩小但不归零)、tanh(压缩到[-1,1]) |
| **Dropout** | 训练时随机"关掉"一些神经元，防止死记硬背 | 正则化技术：以概率p随机置零神经元输出，减少过拟合 |
| **损失函数 (Loss)** | 衡量预测值和真实值差距的"扣分"函数 | 训练目标函数，优化器通过最小化loss来指导参数更新 |
| **优化器 (Optimizer)** | 负责"调整参数"让loss越来越小 | 梯度下降的变体：Adam = 带动量的自适应学习率优化器 |
| **学习率 (Learning Rate)** | 每次调整参数的步长 | 控制参数更新幅度的超参数，太大不收敛，太小训练慢 |
| **Epoch** | 把所有训练数据完整看了一遍 | 训练集全部样本完成一次前向+反向传播 |
| **Batch** | 一次塞进GPU的样本数量 | Mini-batch：每次迭代送入模型的一组样本 |
| **反向传播 (Backprop)** | 算完loss后，倒推每个参数该怎么调 | 链式法则计算损失对每个参数的梯度 |
| **梯度 (Gradient)** | loss对每个参数的"影响程度" | 损失函数对参数的一阶偏导数，指向loss增长最快的方向 |

### 3.2 GAN 相关

| 术语 | 通俗解释 | 专业定义 |
|------|---------|---------|
| **GAN (生成对抗网络)** | 两个AI互相博弈：一个造假，一个识别假货，越练越强 | 由生成器G和判别器D组成的对抗框架，G从噪声/条件生成样本，D区分真伪 |
| **生成器 (Generator)** | "造假者"，输入条件，输出预测图 | 学习从条件分布到目标分布的映射，通过对抗训练欺骗判别器 |
| **判别器 (Discriminator)** | "鉴别师"，判断输入是真是假 | 二分类器，输出patch-level的真假概率图（PatchGAN） |
| **条件GAN (cGAN)** | 给生成器和判别器加"条件"（如几何图），不是瞎生成 | 在G和D的输入中加入条件信号，使生成过程可控 |
| **对抗损失 (GAN Loss)** | "造得够不够像真的"的分数 | 二元交叉熵：G想让D把假样本判为真，D想区分真假 |
| **L1 损失** | "每个像素颜色差了多少"的绝对值平均 | Mean Absolute Error (MAE)，衡量预测图与真实图的像素级差距 |
| **PatchGAN** | 判别器不像人一样看整张图，而是像拿放大镜一块块看 | 全卷积判别器，输出N×N矩阵，每个元素对应输入的一个patch的真假判断 |

### 3.3 网络架构相关

| 术语 | 通俗解释 | 专业定义 |
|------|---------|---------|
| **U-Net** | 先压缩(编码)再放大(解码)，像U形，过程中保留细节 | 编码器-解码器结构，通过跳跃连接将编码层特征拼接到对应解码层 |
| **U-Net 3+** | U-Net的第三代：每个解码层不只看对应编码层，而是看所有编码层 | 全尺度跳跃连接(Full-scale Skip Connections)：每个解码层连接所有编码层和所有更浅的解码层 |
| **编码器 (Encoder)** | 逐步压缩图片、提取高级语义的下采样路径 | 空间分辨率递减、通道数递增的特征提取骨干网络 |
| **解码器 (Decoder)** | 逐步放大、恢复细节的上采样路径 | 空间分辨率递增、结合跳跃连接恢复空间信息的生成网络 |
| **跳跃连接 (Skip Connection)** | 把浅层细节直接传给深层，如同"抄近道" | 将编码器特征图与解码器特征图按通道拼接，融合多尺度信息 |
| **特征图 (Feature Map)** | 卷积层的输出，"学到了什么特征"的图 | 输入经过卷积操作后产生的多通道张量，通道数=filters |
| **通道 (Channel)** | 特征图的"层数"，RGB图有3个通道 | 张量的深度维度，每个通道代表一种特征模式 |
| **感受野 (Receptive Field)** | 一个神经元"看得到"的输入图范围 | 输出特征图上每个像素对应输入图像的空间范围大小 |

### 5.3 数据预处理（第71-103行）

| 操作 | 函数 | 作用 |
|------|------|------|
| 缩放 | `resize()` | 统一图片大小到256×256 |
| 随机裁剪 | `random_crop()` | 先放大到286×286再随机裁256×256，增加多样性 |

### 5.4 数据集构建（第105-119行）

`tf.data.Dataset` 是 TensorFlow 的高性能数据流水线：

### 5.5 网络组件（第121-169行）

这些是搭建整个网络用的"乐高积木"：

| 组件 | 结构 | 用在哪里 |
|------|------|---------|
| `downsample` | Conv2D → BatchNorm → LeakyReLU | 编码器，逐步压缩图片 |
| `upsample` | Conv2DTranspose → BatchNorm → (Dropout) → ReLU | 解码器，逐步放大特征图 |
| `downpool` | MaxPooling2D → Conv2D → BatchNorm → LeakyReLU | U-Net 3+ 的全尺度连接 |
| `upsample2` | UpSampling2D(双线性插值) → Conv2D → BatchNorm → LeakyReLU | 全尺度连接中的快速上采样 |
| `conv` | Conv2D(filters×5) → BatchNorm → LeakyReLU | **注意：这里是 filters×5！** 288→1440通道 |

> **为什么 `conv` 函数要乘以5？**
> `conv(filters)` 里面的 `filters*5` 意味着实际通道数是输入参数的5倍。例如 `conv(288)` 实际输出 288×5=1440 个通道。这是为了让拼接后的特征图有足够的通道去融合多尺度信息。这个设计也导致模型参数量非常大。

### 5.6 Generator（生成器，第172-247行）

```
输入: [几何结构图(256×256×3), 能量场图(256×256×3)]
       ↓ 拼接成 256×256×6
       ↓ 逐层下采样（编码）
256 → 128 → 64 → 32 → 16 → 8 → 4 → 2 → 1  (空间分辨率)
 6     32   64  128  128  256  256  512  512  (通道数)
       ↓ 全尺度跳跃连接 + 逐层上采样（解码）
 1  →  2  →  4  →  8  → 16 → 32 → 64 → 128 → 256
512   融合  融合  融合  融合  融合  融合  融合   融合
       ↓ 最后卷积层(tanh激活)
输出: [裂纹预测图(256×256×3)]
```

**U-Net 3+ 的核心创新——全尺度跳跃连接**：

普通 U-Net 只在同层级连接（编码层2 → 解码层2）。U-Net 3+ 的每个解码层连到**所有**编码层，还连到**所有更浅的解码层**。例如解码层 `xd4`（32×32）连接了：
- x41: 来自 xe1 (256×256) 池化到32×32
- x42: 来自 xe2 (128×128) 池化到32×32
- x43: 来自 xe3 (64×64) 池化到32×32
- x44: 来自 xe4 (32×32) 直接卷积
- x45: 来自 xd5 (16×16) 上采样到32×32
- x46: 来自 xd6 (8×8) 上采样到32×32
- x47: 来自 xd7 (4×4) 上采样到32×32
- x48: 来自 xd8 (2×2) 上采样到32×32
- x49: 来自 xe9 (1×1) 上采样到32×32

共9条连接，每条贡献32个通道，拼接后32×9=288通道。

**参数量：53,804,291（约5400万）**

### 5.7 Discriminator（判别器，PatchGAN，第250-285行）

```
输入: [几何结构图, 能量场图, 裂纹图(真或假)]
       ↓ 拼接成 256×256×9 (3+3+3)
       ↓ 3层下采样
256 → 128 → 64 → 32
 9     64   128  256
       ↓ ZeroPadding + Conv
31 × 31 × 512
       ↓ BatchNorm + LeakyReLU
       ↓ 最终卷积 → 1通道
输出: 30×30 的"真假判断矩阵"
```

**PatchGAN 的含义**：不是输出"整张图是真是假"的一个值，而是输出一个30×30的矩阵，每个元素判断对应 patch（感受野约70×70像素）是真是假。这比全局判断能更好地捕捉局部细节。

**参数量：2,773,505（约280万）**

### 5.8 损失函数（第287-300行）

```
总损失 = GAN损失 + λ × L1损失
        = GAN损失 + 100 × 像素平均差异
```

- **GAN损失** (`BinaryCrossentropy`)：生成器想让判别器把假样本判定为真（标签=1），判别器想区分真假
- **L1损失** (λ=100)：逐像素比较预测裂纹和真实裂纹的绝对差异。λ=100 意味着像素精度比"骗过判别器"重要100倍

```python
generator_optimizer = Adam(lr=1e-4, beta_1=0.5)    # 生成器的学习率
discriminator_optimizer = Adam(lr=1e-5, beta_1=0.5) # 判别器学习率更低（原版论文传统）
```

### 5.9 检查点系统（第302-318行）

```python
generator.save_weights(gen_ckpt_path)     # 保存权重
generator.load_weights(gen_ckpt_path)     # 加载权重
```

使用 TF2 的 `.weights.h5` 格式，只保存模型权重（不含结构）。配合 `Loss.txt` 实现**断点续训**：中途停止后重新运行，会自动从上次中断的 epoch 继续。

### 5.10 训练循环（第321-376行）

每个 epoch 的流程：

```
1. 生成1张预览图 → Out_rock/epoch_XXXX_1.png
2. 遍历训练集（1600张/1 = 1600步）
   每步：前向 → 算loss → 反向传播 → 更新参数
3. 记录loss到 Loss.txt
4. 保存检查点 → SaveModel/
```

`tf.GradientTape()` 是 TF2 的自动微分机制，"录制"前向计算过程，然后自动计算梯度。

## 6. 我做的修改清单

### 原始代码 (Main code.py) 的 6 个致命 Bug

| 编号 | 问题 | 原因 | 后果 |
|------|------|------|------|
| 🔴 1 | **TF1.x 兼容模式** | 使用了 `tensorflow.compat.v1.ConfigProto`、`InteractiveSession` | TF 2.x 上运行异常 |
| 🔴 2 | **Generator 输入维度不匹配** | 定义时只有1个输入 `inp`，调用时却传2个 `[input_image, energy_image]` | 直接报错 |
| 🔴 3 | **Discriminator 输入维度不匹配** | 定义时只有2个输入 `[inp, tar]`，调用时传了3个 `[input, energy, target]` | 直接报错 |
| 🔴 4 | **数据文件名错误** | 代码里写 `doubleTrain_*.png`，实际文件是 `Train_*.jpg` | 找不到任何训练数据 |
| 🔴 5 | **数据路径错误** | 路径 `./Train_images/`，实际是 `./Train_images/Train_images/` | 找不到图片 |
| 🔴 6 | **`if` 在 Graph 模式下使用** | `random_jitter()` 里的 `if tf.random.uniform(()) > 0.5:` 在 TF2 的 AutoGraph 中不合规 | `OperatorNotAllowedInGraphError` 崩溃 |

### 修改内容详解

#### 修改 1：移除 TF1.x 兼容代码

```python
# ❌ 旧代码（TF1 兼容，不好用）
from tensorflow.compat.v1 import ConfigProto
from tensorflow.compat.v1 import InteractiveSession
config = ConfigProto()
config.gpu_options.per_process_gpu_memory_fraction = 0.8
session = InteractiveSession(config=config)

# ✅ 新代码（TF2 原生方式）
gpus = tf.config.list_physical_devices('GPU')
if gpus:
    for gpu in gpus:
        tf.config.experimental.set_memory_growth(gpu, True)
```

**效果**：代码不再依赖 TF1 兼容层，用 TF2 原生的内存增长模式代替按比例分配。

#### 修改 2：修复 Generator 输入（核心修复）

```python
# ❌ 旧代码：只有1个输入，但调用时传2个值

xe1 = inp                            # ← 只有 inp，没能量场

# ✅ 新代码：正确接收2个输入并拼接
inp = tf.keras.layers.Input(..., name='input_image')
```

### 8.2 GPU 问题说明

⚠️ **TensorFlow 2.11+ 在原生 Windows 上不再支持 GPU 加速！**

你的 TF 2.21 只能用 CPU。要启用 GPU，有两个方案：

| 方案 | 难度 | 性能 |
|------|------|------|
| **WSL2** | ⭐⭐ (需要装子系统) | ⭐⭐⭐⭐⭐ 满血GPU |
| **DirectML 插件** | ⭐ (装个包就行) | ⭐⭐⭐ 部分加速 |
| **CPU only** (当前) | 无 | ⭐ 很慢 |

推荐装 WSL2，把代码放到 WSL2 里跑。这是官方推荐的 Windows 深度学习方案。

## 9. FAQ

### Q1: 我什么都不会，能跑通吗？

A: 能。只用在终端里输入 `python "Main code.py"` 就行。所有复杂的东西代码自己处理。

### Q2: 输入图和输出图长什么样？

A: 查看 `Out_rock/` 目录下的预览图。每张4列：几何输入 | 能量输入 | 标准答案裂纹 | AI预测裂纹。

### Q3: 怎么判断训练好了没有？

A: 看两样：
1. `Loss.txt` 里的数字整体在下降
2. `Out_rock/` 里最新的预览图，AI预测的裂纹和标准答案越来越像

### Q4: 训练到一半电脑太烫/太慢怎么办？

A: `Ctrl+C` 停止，检查点已自动保存。下次有空再跑会自动继续。

### Q5: 想把结果给别人看怎么办？

A: `Out_rock/` 里的 PNG 图就是预测结果，可以直接发。

### Q6: Loss 不降反升怎么办？

A: 正常现象，GAN 训练会波动。看整体趋势即可，偶尔的上升不用担心。
